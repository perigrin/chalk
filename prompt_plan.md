## Prompt 2: XSUB Node + CompositeNode (PREINIT/CODE Partitioning) ✅ COMPLETED

**Milestone**: 5.0 (XS AST Infrastructure — part 2, completing milestone)

**Depends on**: Prompt 1

```text

Read docs/xs-target-spec.md section 3.1 for XSUB partitioning details.

Add tests to the existing t/bootstrap/xs-ast.t, then implement:

1. **Chalk::Bootstrap::Target::XS::AST::CompositeNode** (lib/Chalk/Bootstrap/Target/XS/AST/CompositeNode.pm)
   - :isa(Chalk::Bootstrap::Target::XS::AST::Node)
   - Field: $children :param :reader (arrayref of AST nodes)
   - emit() calls emit() on each child in order and concatenates results

2. **Chalk::Bootstrap::Target::XS::AST::XSUB** (lib/Chalk/Bootstrap/Target/XS/AST/XSUB.pm)
   - :isa(Chalk::Bootstrap::Target::XS::AST::Node)
   - Fields: $return_type :param :reader (default 'SV *'),
             $name :param :reader,
             $params :param :reader (arrayref of strings like 'SV *self'),
             $body :param :reader (arrayref of mixed VarDecl and Statement nodes)
   - emit() partitions $body: VarDecl nodes go to PREINIT, everything else
     goes to CODE. Output format:

     ```
     SV *
     Grammar(self)
         SV *self
       PREINIT:
         AV *expressions;
         SV *symbol;
       CODE:
         expressions = newAV();
         { dSP; ... }
         RETVAL = rule;
       OUTPUT:
         RETVAL
     ```

   - PREINIT section is only emitted if there are VarDecl nodes
   - Parameters are indented 4 spaces under the function signature
   - PREINIT/CODE/OUTPUT labels are indented 2 spaces
   - Body content is indented 4 spaces

Tests should verify:
- CompositeNode emits children in order
- CompositeNode with empty children emits empty string
- XSUB with only Statements (no VarDecls) omits PREINIT section
- XSUB with mixed VarDecl + Statement correctly partitions them
- XSUB emits correct signature line (return_type + name + params)
- XSUB always emits OUTPUT: RETVAL section
- Full XSUB with realistic content matches expected XS structure

Run tests with: perl -Ilib t/bootstrap/xs-ast.t
Commit when all tests pass.
```

## Prompt 6: Symbol Construction Lowering ✅ COMPLETED

**Milestone**: 5.1 (Target::XS Graph Visitor — part 3)

```text
Add Constructor:Symbol lowering to Target::XS. This is the core XS code
generation pattern — building a call_method("new") block that constructs a
Chalk::Grammar::Symbol from XS.

Read docs/xs-target-spec.md sections 2.3, 3.2, and 5.2 for the construction
pattern and variable naming.

Each Symbol construction produces:
1. A VarDecl node for the result variable (SV *sym_N)
2. A Statement node containing the full call_method block

The call_method block for a Symbol with quantifier looks like:
```c
{
    dSP;
    ENTER; SAVETMPS;
    PUSHMARK(SP);
    XPUSHs(sv_2mortal(newSVpvs("Chalk::Grammar::Symbol")));
    XPUSHs(sv_2mortal(newSVpvs("type")));
    XPUSHs(sv_2mortal(newSVpvs("reference")));
    XPUSHs(sv_2mortal(newSVpvs("value")));
    XPUSHs(sv_2mortal(newSVpvs("Rule")));
    XPUSHs(sv_2mortal(newSVpvs("quantifier")));
    XPUSHs(sv_2mortal(newSVpvs("+")));
    PUTBACK;
    call_method("new", G_SCALAR);
    SPAGAIN;
    sym_0 = SvREFCNT_inc(POPs);
    PUTBACK;
    FREETMPS; LEAVE;
}
```

When quantifier is undef, omit the quantifier key-value pair entirely.

Using TDD, add tests to xs-target.t then implement:

1. **method _emit_symbol($symbol_node, $var_name)**:

   - Handles optional quantifier (include args only when defined)
   - Uses _escape_c_string() and _strip_terminal_delimiters() from Prompt 5

2. **Symbol counter** — Track a counter field to generate sym_0, sym_1, etc.
   Reset per XSUB.

Test cases (build IR with NodeFactory, same helpers as codegen-perl.t):
- Reference symbol without quantifier → block has type+value args, no quantifier
- Terminal symbol → value has delimiters stripped and C-escaped
- Symbol with quantifier → block includes quantifier key-value pair

## Prompt 7: Expression + Rule Construction Lowering ✅ COMPLETED

**Milestone**: 5.1 (Target::XS Graph Visitor — part 4)
**Files modified**: Target/XS.pm, xs-target.t
**Depends on**: Prompt 6

```text
Add Constructor:Expression and Constructor:Rule lowering to Target::XS.

Read docs/xs-target-spec.md sections 5.2-5.3 for the lowering patterns.

**Expression lowering** produces:
- A VarDecl: AV *expr_N
- A Statement: expr_N = newAV();
- For each symbol: the symbol's VarDecl + Statement (from _emit_symbol)
- For each symbol: a Statement: av_push(expr_N, sym_M);

**Rule lowering** produces:
- VarDecls for: AV *expressions, SV *rule (per XSUB)
- A Statement: expressions = newAV();
- For each expression: expression's VarDecl + Statements
- For each expression: Statement: av_push(expressions, newRV_noinc((SV *)expr_N));
- The call_method("new") block for Rule construction, including
  newRV_noinc((SV *)expressions) as the expressions argument

The Rule call_method block:
```c
{
    dSP;
    ENTER; SAVETMPS;
    PUSHMARK(SP);
    XPUSHs(sv_2mortal(newSVpvs("Chalk::Grammar::Rule")));
    XPUSHs(sv_2mortal(newSVpvs("name")));
    XPUSHs(sv_2mortal(newSVpvs("Grammar")));
    XPUSHs(sv_2mortal(newSVpvs("expressions")));
    XPUSHs(sv_2mortal(newRV_noinc((SV *)expressions)));
    PUTBACK;
    call_method("new", G_SCALAR);
    SPAGAIN;
    rule = SvREFCNT_inc(POPs);
    PUTBACK;
    FREETMPS; LEAVE;
}
```

Note: the expressions argument uses newRV_noinc (not newSVpvs) because
it's a reference to the AV, not a string.

Using TDD, add tests to xs-target.t then implement:

1. **method _emit_expression($expr_node, $var_name)**:
   - Returns arrayref of AST nodes (VarDecl + Statements)
   - Creates the AV, emits each symbol, pushes each onto the AV

2. **method _emit_rule($rule_node)**:
   - Returns arrayref of AST nodes (all VarDecls + Statements for the rule)
   - Creates expressions AV, emits each expression, pushes each as ref
   - Emits Rule call_method block
   - Adds RETVAL = rule; as final Statement
   - Resets symbol/expression counters

Test cases:
- Single-symbol expression → 1 VarDecl (expr), 1 Statement (newAV),
  symbol nodes, 1 Statement (av_push)
- Multi-symbol expression → multiple symbols and av_pushes
- Single-alternative rule → 1 expression, 1 av_push to expressions
- Multi-alternative rule → N expressions, N av_pushes
- Full rule output includes RETVAL assignment
- Rule name is properly C-escaped in the call_method block

Run: perl -Ilib t/bootstrap/xs-target.t
Commit when all tests pass.
```
