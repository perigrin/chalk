## Executive Summary

Three targeted codegen fixes for Tier D XS compilation. The fixes are correct and well-tested, recovering 4 previously failing tests. ## Critical Issues

### [C1] Exception-unsafe save/restore of `$_current_sub_name` in `_emit_sub`
- **File:** `lib/Chalk/Bootstrap/Perl/Target/C.pm:336-437`
- **Bug:** `_emit_sub` sets `$_current_sub_name` at entry (line 338) and restores at exit (line 437). Both callers wrap `_emit_sub` in `eval {}` (lines 1450, 1468). If the body throws between set and restore, `$_current_sub_name` leaks the dead sub's name. Subsequent method compilation would falsely trigger `__SUB__` recursion detection (EmitHelpers.pm:1749), generating direct C calls to a nonexistent helper function.
- **Impact:** Silent miscompilation — either linker error or wrong-function call at runtime
- **Suggested fix:** Wrap body in eval and always restore, or use `defer { $self->_set_current_sub_name($prev_sub_name); }` (available in Perl 5.42.0)
- **Confidence:** High
- **Found by:** Logic, Error Handling, Contract, Concurrency (4/5 specialists)

## Important Issues

### [I1] False positive `__SUB__` recursion for any undef-target coderef call in my-sub body
- **File:** `lib/Chalk/Bootstrap/Perl/Target/EmitHelpers.pm:1749-1750`
- **Bug:** Detection `!defined $target && length $_current_sub_name` treats ANY undef-target coderef call inside a `my sub` as `__SUB__` recursion. If the parser loses a different coderef target (e.g., `$callback->($arg)`) to stale-value merge, the wrong function gets called.
- **Impact:** Silent wrong codegen if any non-`__SUB__` coderef call has its target lost to parser merge
- **Suggested fix:** Add a comment documenting the assumption, or tag `__SUB__` in the IR for positive detection
- **Confidence:** Medium
- **Found by:** Logic, Error Handling, Contract (3 specialists)

### [I2] `/_ADJUST$/` suffix match could collide with user method names
- **File:** `lib/Chalk/Bootstrap/Perl/Target/C.pm:1705`
- **Bug:** A method named `do_ADJUST` would produce exported function `slug_do_ADJUST` matching `/_ADJUST$/`, incorrectly excluding it from XSUBs and treating it as the ADJUST block. Same risk for `/_init_statics$/`.
- **Impact:** User method silently vanishes from XS interface
- **Suggested fix:** Anchor with slug: `$fn->{name} =~ /^\Q${slug}\E_ADJUST$/`
- **Confidence:** Medium
- **Found by:** Logic, Error Handling (2 specialists)
