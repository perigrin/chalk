## Critical Issues

### [C1] Multi-phi stale incoming labels in _wire_region_phis_with_preblock
- **File:** `lib/Chalk/Target/LLVM.pm:5496-5530`
- **Bug:** Phis are wired one at a time; phi N is emitted with the tail
  labels current at its iteration. If phi N+1's arm is multi-block,
  `_lower_arm_in_tail` consumes that tail's br-to-merge and re-attaches it
  to a continuation block — phi N's already-emitted line now names a
  non-predecessor. Live-reproduced: Region with PhiA=[5,7] and
  PhiB=[Subscript, 9] → lli "PHI node entries do not match predecessors!".
- **Impact:** Invalid IR whenever a Region has ≥2 explicit Phi consumers and
  a later one has a multi-block arm.
- **Fix:** Two-pass — lower ALL arms for ALL phis first (collecting values +
  final labels), then emit all phi lines with the final labels. (The loop
  fix in commit C already used this batch shape for backedges.)
- **Confidence:** High
- **Found by:** Logic, Edge-cases, Contract (independent)
- **DISPOSITION: FIX NOW** (in-range code).

## Important Issues

### [I1] /#\d+$/ per-call discriminator false-positives on content-hash ids
- **File:** `lib/Chalk/IR/Graph.pm:57, 76, 142`
- **Bug:** Content-hash ids embed input ids; any node whose LAST serialized
  input is per-call matches (`PadAccess|targ=0|varname=$x|VarDecl#3`,
  `Negate|Phi#2`), as does a string Constant whose value ends in `#<digits>`.
  Such nodes take the per-call branch: merge() stops canonicalizing
  content-equal distinct objects (returns/overwrites with the handed node).
- **Impact:** Latent today (per-parse factory dedups first, id ==
  content_hash so the cache key coincides) but the predicate is wrong and
  diverges for nodes built outside the factory (deserialized graphs, _seed,
  test-built nodes).
- **Fix:** Per-call ids never contain `|`: `$id !~ /\|/ && $id =~ /#\d+$/`.
- **Confidence:** High (spurious match certain; harm latent)
- **Found by:** Logic, Edge-cases, Contract (independent)
- **DISPOSITION: FIX NOW** (in-range code).

### [I2] Missing undef-arm guard in _wire_region_phis_with_preblock
- **File:** `lib/Chalk/Target/LLVM.pm:5505-5517`
- **Bug:** Bindings builds `Phi([then_val, undef])` when a var is bound in
  one branch only; the wiring passes the undef to lower_value, which crashes
  with "Can't call method id on undefined value" instead of a diagnostic.
  `_lower_phi` has the guard (H4/H5); this rewrite touched the exact lines
  without porting it.
- **Fix:** Per-slot defined check with a loud GAP die before lowering arms.
- **Confidence:** Medium
- **Found by:** Edge-cases
- **DISPOSITION: FIX NOW** (cheap, same function as C1).
